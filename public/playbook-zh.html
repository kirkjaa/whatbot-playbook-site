设置与长期上下文

开始正式工作前，先选择正确的 profile，保存可复用的事实，并确认 workspace、model、skills 与 MCP 配置。

MemorySkillsMCP/A2A SettingsModel Configuration
Agent Profile Selector当前 run 完成后切换 profile；UI 会在该 chat 中保存选择。
Memory保存公司事实、品牌语气、投资人背景与偏好，供后续对话复用。
Skills把项目 skills 固定到 prompt extras，让专业框架在整个 workflow 中持续生效。
上下文设置 Prompt 
请把以下内容记为长期 venture context：

公司：[公司名称]
一句话描述：[产品、客户、解决的问题]
阶段：[Idea / MVP / Revenue / Growth / Fundraising]
目标市场：[ICP + 地区]
商业模式：[SaaS / marketplace / services / licensing / hybrid]
竞争对手：[3-5 个竞争对手]
当前指标：[MRR、用户、试点、留存、CAC、runway]
融资目标：[轮次金额、时间、资金用途]
首选输出语言：中文。

除非我更新这些信息，否则之后的分析都使用这份上下文。
最佳实践：只把稳定事实写入 memory；假设、实验方向和临时讨论保留在当前 chat context 中。

选择合适的 Agent Profile

v1.7 将 Whatbot 的业务 profiles 与专业工具结合。先选择 lead agent，再在需要时委派给专家 profile。

Venture Builder创意验证、市场研究、竞争分析、融资准备、增长与退出策略。
Financial Analyst通过 Fincept Terminal 做 CFA 级估值、组合、衍生品、风险、量化与回测。
Marketing Strategist25 个技能覆盖 CRO、文案、SEO、广告、邮件、定价、launch 与增长。
Researcher研究、数据分析、引用与结构化报告。
Developer软件开发、架构、debug 与代码执行 workflow。
Hacker安全审查、渗透测试、风险识别与加固建议。
Profile 选择 Prompt 
我想运行完整的 venture-building workflow。

请用 Venture Builder 作为 lead profile。必要时委派：
- 市场与来源研究给 Researcher
- 财务建模给 Financial Analyst
- GTM 与 messaging 给 Marketing Strategist
- 技术可行性给 Developer
- 安全与风险检查给 Hacker
- 文档生成给 QUEST Studio

请保持单一共享上下文，并在切换 agent/profile 能提升质量时提醒我。

运行 Venture Builder Pipeline

核心 `venture_pipeline` 提供 7 阶段 venture-build 方法，支持语言参数，可生成研究分析、图片 prompts、HTML 输出和下载链接。

Venture Builderventure-buildHTML OutputsSearch + Images
Startup Brief→Market Discovery→Gap Analysis→Value Prop→Risk & Q&A→HTML Venture Brief
阶段 1-2竞争者、融资、TAM/SAM/SOM、目标客群、痛点、功能 gap 与蓝海机会。
阶段 3-5USP、persona、定位语句、SWOT、风险矩阵、可行性评分与投资人 Q&A。
阶段 6-7HTML venture brief 与互动 market landscape，包括 2x2 定位和功能对比。

请用中文运行完整 Venture Builder pipeline。

Venture name: [公司]
Description: [3-5 句说明产品、目标客户、问题、解决方案]
Industry: [行业 / vertical]
Target market: [ICP + 地区]
Competitors: [竞争对手 1、2、3]
Team background: [创始人/团队优势]

生成：
1. 市场与竞争发现
2. 产品 gap 分析
3. 价值主张与定位
4. SWOT、风险矩阵与可行性评分
5. 投资人和客户 Q&A
6. HTML venture brief
7. HTML market landscape

最终回复中请完整保留所有 download links。

使用多智能体 Venture Consulting

可以把问题路由给单一专家、智能选择 1-3 位分析师，或用 turbo mode 同时运行 5 位专家做 board-level review。

venture-consultantMarket AnalystBusiness AnalystCompetitor AnalystFinancial AnalystPitch Consultant
Smart Mode推荐默认模式：选择最相关的 1-3 位专家。
Turbo Mode并行运行五位专家，获得全面覆盖。
Single-Agent Mode使用 `market_analyst`、`business_analyst`、`competitor_analyst`、`financial_analyst` 或 `pitch_analyst`。
Turbo Consulting Prompt 
请用 turbo mode 运行 Venture Consultant review。

Context:
[公司、产品、traction、商业模式、市场、融资目标]

请五位专家分别分析：
1. 市场规模、增长与时机
2. 商业模式与定价
3. 竞争定位与 moat
4. 财务预测、unit economics、runway
5. Pitch narrative、投资人 objections 与融资 readiness

最后给出优先级排序的 30 / 60 / 90 天行动计划。

Investor Deck Prompt 
请使用 QUEST Studio `generate_doc`。

Document type: pitch_deck
Quality: hideft
Audience: Seed / Series A investors
Topic:
[公司概况、产品、traction、市场、商业模式、融资金额、资金用途]

Style direction:
[品牌颜色、视觉情绪、参考素材、限制条件]

交付、分支与迭代

把第一次运行转成可复用的 venture package，再通过 chat branches 探索不同策略，同时保留主线。

Chat BranchingChat CompactionTime TravelDocument PreviewInfection Check
Investor PackagePitch deck、one-pager、executive summary、financial model、investor Q&A。
Operating PackageGTM calendar、landing copy、email sequence、analytics plan、product roadmap。
Sales PackageSales deck、case study、battle cards、objection scripts、comparison chart。
Governance PackageBoard deck、investor update、risk review、security checklist、data room outline。
最终 Package Prompt 
请把本 chat 的所有成果整理成最终 venture operating package。

包含：
1. Executive summary
2. Venture brief
3. Market and competitive analysis
4. Financial model summary
5. GTM strategy
6. Investor deck checklist
7. Open risks and assumptions
8. 30 / 60 / 90 day execution plan
9. File/download link inventory

然后建议三个有用的 chat branches：
- conservative financial case
- aggressive growth case
- skeptical investor diligence case
迭代循环：用 branches 对比方案，长对话稳定后 compact，用 time travel 检查 workspace diff，并把最终文档保存在 document preview 或 QUEST Studio 中导出。
